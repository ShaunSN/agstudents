1. Introduction to the Tutorial

What is KoboToolbox?

KoboToolbox is a free, open-source platform designed for data collection in challenging environments, such as agricultural research, humanitarian aid, and public health. It supports digital surveys deployable via mobile devices or web browsers, with offline capabilities and multilingual support. This tutorial focuses on the web interface, a user-friendly form builder suitable for beginners and advanced users alike.

Purpose of the Tutorial

This tutorial guides you through creating a comprehensive broiler farmer survey based on the Broiler Farmer Questionnaire. By the end, you’ll be able to:

  • Set up a project and form in KoboToolbox.
  • Add various question types (e.g., text, integer, select_one, select_multiple).
  • Organize questions into logical groups for better survey flow.
  • Implement skip logic, constraints, and calculations for dynamic surveys.
  • Preview, test, and deploy the survey for data collection.

Prerequisites

To follow this tutorial, you need:

  • A KoboToolbox account (sign up at kobotoolbox.org).
  • Basic understanding of survey design concepts (e.g., question types, skip logic).
  • Access to a web browser for the KoboToolbox web interface.

Context

The survey collects data on broiler farmers’ demographics, general background, chicken rearing practices, feeding strategies, nutritional knowledge, economic considerations, challenges, performance outcomes, and future improvement suggestions. This data is critical for agricultural research and support programs.

2. Setting Up Your Project

Creating a New Project

Start by setting up a project in KoboToolbox to house your survey form.

Instructions

  1. Log In: Go to kobotoolbox.org and sign in with your credentials.
  2. Start a Project: On the dashboard, locate the + New Project button and click it.
  3. Name the Project: Enter Broiler Farmer Survey as the project name to clearly identify it.
  4. Create Project: Click Create Project to initialize the project. You’ll be taken to the project dashboard.
  5. Verify: Ensure the project appears in your dashboard with the correct name.

Opening the Form Builder

The form builder is where you’ll design the survey. It’s a visual interface with tools to add questions, groups, and configure settings.

Instructions

  1. Create a Form: In the project dashboard, select Build from scratch to open the web-based form builder.
  2. Understand the Interface:
    • Sidebar (Left): Contains options to add questions, groups, or import XLSForms.
    • Main Area (Center): Displays the form structure, where you can view and edit questions.
    • Settings Panel (Right): Allows configuration of question properties, such as required fields, skip logic, and constraints.
  3. Save Progress: Click the save icon (floppy disk) in the top toolbar periodically to avoid losing changes.
  4. Test Navigation: Click through the sidebar and main area to familiarize yourself with adding and editing questions.

Tip: Save your work frequently using the save icon to prevent data loss, especially during long editing sessions.

3. Understanding Question Types

Overview

KoboToolbox offers various question types to capture different data formats. Choosing the correct type ensures data is collected in a format suitable for analysis.

  • Text: For open-ended responses (e.g., respondent’s name).
  • Integer: For whole numbers (e.g., age in years).
  • Decimal: For numbers with decimals (e.g., feed cost in PGK).
  • Select One: For single-choice questions (e.g., Gender: Male or Female).
  • Select Multiple: For multiple-choice questions (e.g., reasons for choosing farming).
  • Geopoint: For capturing GPS coordinates (e.g., farm location).
  • Calculate: For computing values based on other inputs (e.g., profit).
  • Note: For non-data text, such as instructions or context.

Choosing the Right Type

Select question types based on the data you need:

  • Use select_one for mutually exclusive options to ensure single responses.
  • Use integer or decimal for numerical data suitable for calculations.
  • Use text for qualitative, descriptive responses.
  • Use geopoint for spatial data, critical for agricultural surveys.

Examples

Here’s how question types apply to the survey:

  • Question: How old are you? Type: integer (Reason: Age is a whole number, e.g., 45).
  • Question: Normal market price per live chicken? Type: decimal (Reason: Monetary values may include decimals, e.g., 25.50 PGK).
  • Question: Why did you choose broiler farming? Type: select_multiple, Choices: Cash (Income), Low risk, Quick turnover (Reason: Allows multiple reasons).

4. Adding Questions to the Survey

This section details how to build the survey’s sections (Demographics, General Background, Chicken Background, Feeding Practices, Nutritional Knowledge, etc.) in KoboToolbox, including question types, skip logic, and constraints.

4.1 Demographics Section

Create Group: Organize demographic questions into a group for clarity.

  • Name: demographics
  • Label: Demographics

Add Questions: Add the following questions within the group.

  • Name:
    • Type: text
    • Label: Name
    • Variable Name: name
    • Hint: Enter full name of respondent
  • Age:
    • Type: integer
    • Label: Age
    • Variable Name: age
    • Hint: Enter age in years
  • Place of Origin:
    • Type: text
    • Label: Place of Origin (Home Province)
    • Variable Name: origin
  • Address:
    • Type: text
    • Label: Current address (Home residence/Location)
    • Variable Name: address
  • Gender:
    • Type: select_one gender
    • Label: Gender
    • Variable Name: gender
    • Choices: Male, Female
  • Marital Status:
    • Type: select_one marital_status
    • Label: Marital Status
    • Variable Name: marital_status
    • Choices: Married, Single, Divorced, Defacto, Rather not say
  • Occupation:
    • Type: select_one occupation
    • Label: Main Occupation
    • Variable Name: occupation
    • Choices: Paid Employment, Home Carer, Chicken Farmer, Other
  • Currently Raising:
    • Type: select_one yes_no
    • Label: Are you currently raising broilers?
    • Variable Name: currently_raising
    • Choices: Yes, No
  • Stopped Raising:
    • Type: select_one yes_no
    • Label: Have you raised broilers but stopped?
    • Variable Name: stopped_raising
    • Choices: Yes, No

Instructions

  1. Create Group: In the form builder, click + Add Group in the sidebar. Set Name to `demographics` and Label to `Demographics`. Save the group.
  2. Add Questions: For each question:
    • Click + Add Question within the `demographics` group.
    • Select the appropriate Type (e.g., `text` for Name, `integer` for Age, `select_one` for Gender).
    • Enter the Label exactly as listed (e.g., "Name").
    • Set the Variable Name (e.g., `name`, `age`) for data export and reference in logic.
    • For `select_one` questions, add Choices in the Choices tab (e.g., for Gender, add "Male" and "Female").
    • Add Hint where specified (e.g., "Enter full name of respondent" for Name) in the Settings panel.
    • Optional: Add Constraint for Age (e.g., `${age} >= 18` to ensure respondents are adults) in the Constraints field.
  3. Order Questions: Drag questions in the main area to match the order: Name, Age, Place of Origin, Address, Gender, Marital Status, Occupation, Currently Raising, Stopped Raising.
  4. Test Question Flow:
    • Click Preview (eye icon) in the form builder.
    • Enter sample responses (e.g., "John Doe" for Name, 35 for Age, "Male" for Gender).
    • Verify that questions display in order, choices for `select_one` (e.g., Gender, Marital Status) show all options, and text/integer inputs accept valid data.
    • Check that the group label "Demographics" appears at the top of the section in Preview.
    • If Age constraint is added, try entering 17; an error should prevent submission.
  5. Save: Click the save icon to store changes.

Tip: Use consistent variable names (e.g., `name`, not `Name`) to avoid errors in data export or skip logic later. Add hints to clarify inputs for respondents.

4.2 General Background Section

Create Group: Group questions about the farmer’s background and motivations.

  • Name: general_background
  • Label: Section 1: General Background

Add Questions:

  • Farm Location:
    • Type: geopoint
    • Label: 1.1. Where are you located (location of farm)?
    • Variable Name: farm_location
    • Hint: Record GPS coordinates of the farm
  • Years Raising:
    • Type: integer
    • Label: 1.2. How long have you been raising broiler chickens?
    • Variable Name: years_raising
    • Hint: Enter number of years
  • Motivation:
    • Type: select_multiple motivation
    • Label: 1.3. Why did you choose broiler farming?
    • Variable Name: farming_motivation
    • Choices: Cash (Income), For special needs, Low risk, Low capital investment, Quick turnover
  • Training Received:
    • Type: select_one yes_no
    • Label: 1.4. Have you undergone training before raising chickens?
    • Variable Name: training_received
    • Choices: Yes, No
  • Training Details:
    • Type: text
    • Label: 1.5. If yes, when and by who?
    • Variable Name: training_details
    • Skip Logic: selected(${training_received}, 'yes')
  • Introduction Source:
    • Type: select_one intro_source
    • Label: 1.6. Who introduced broiler chicken farming to you?
    • Variable Name: introduction_source
    • Choices: Friend, Family member/Relative, Neighbour
    • Skip Logic: selected(${training_received}, 'no')
  • Why Stopped:
    • Type: select_multiple stop_reason
    • Label: 1.7. Why did you stop broiler farming?
    • Variable Name: why_stopped
    • Choices: No access to feed, High prices of feed, No access to day-old chicks, High prices of day-old chicks, No access to markets, Expenses exceeded revenue, Other (explain)
    • Skip Logic: selected(${stopped_raising}, 'yes')

Instructions

  1. Create Group: In the form builder, click + Add Group. Set Name to `general_background` and Label to `Section 1: General Background`. Save.
  2. Add Questions: For each question:
    • Click + Add Question inside the `general_background` group.
    • Select Type (e.g., `geopoint` for Farm Location, `select_multiple` for Motivation).
    • Enter the Label (e.g., "1.1. Where are you located (location of farm)?").
    • Set Variable Name (e.g., `farm_location`).
    • Add Choices for `select_one`/`select_multiple` (e.g., for Motivation, add "Cash (Income)", "For special needs", etc.).
    • Add Hint where listed (e.g., "Record GPS coordinates of the farm" for Farm Location).
    • For questions with Skip Logic:
      • Click the gear icon, go to Skip Logic tab.
      • Add condition (e.g., for Training Details, set `selected(${training_received}, 'yes')`).
    • Add Constraint for Years Raising: Set `${years_raising} >= 0` to allow zero years for new farmers.
  3. Order Questions: Arrange questions in order: Farm Location, Years Raising, Motivation, Training Received, Training Details, Introduction Source, Why Stopped.
  4. Test Skip Logic:
    • In Preview, test Training Details: Set `training_received` to "Yes" (should show), then "No" (should hide).
    • Test Introduction Source: Set `training_received` to "No" (should show), then "Yes" (should hide).
    • Test Why Stopped: Set `stopped_raising` to "Yes" (should show), then "No" (should hide).
  5. Test Question Flow:
    • Enter sample responses (e.g., GPS coordinates for Farm Location, 5 for Years Raising, multiple selections for Motivation).
    • Verify group label "Section 1: General Background" displays.
    • Ensure `select_multiple` (Motivation, Why Stopped) allows multiple selections, and `geopoint` captures coordinates.
    • Test constraint: Try negative value for Years Raising; an error should prevent submission.
  6. Save: Click the save icon.

Tip: Test skip logic thoroughly in Preview to ensure questions appear only when conditions are met, avoiding respondent confusion.

5. Testing and Validation

Why Test?

Testing ensures the survey functions as intended, with correct question flow, skip logic, constraints, and user experience. Validation confirms data is collected accurately for analysis.

Steps to Test the Survey

  1. Preview the Form:
    • In the form builder, click the Preview icon (eye) in the top toolbar.
    • Navigate through the survey, entering sample responses for each question (e.g., "John Doe" for Name, 25.50 for Market Price, select "Yes" for Training Received).
    • Verify group labels (e.g., "Demographics", "Section 3: Feeding Practices") display clearly.
    • Check that all questions appear in the correct order within their groups.
  2. Test Skip Logic:
    • For questions with skip logic (e.g., Training Details, Starter Duration):
      • Test positive conditions (e.g., set `training_received` to "Yes" to show Training Details; set to "No" to hide it).
      • Test negative conditions (e.g., set `starter_used` to "No" or "Own mix" to hide Starter Duration).
      • Confirm questions only appear when their conditions are met (e.g., Different Brand should only show if `same_brand` is "No").
  3. Test Constraints:
    • For questions with constraints (e.g., Age: `${age} >= 18`, Feed Cost: `${feed_cost} > 0`):
      • Enter invalid inputs (e.g., 15 for Age, -5 for Feed Cost) and check that an error message prevents submission.
      • Enter valid inputs (e.g., 30 for Age, 150.50 for Feed Cost) and confirm they are accepted.
  4. Test Question Types:
    • For `text` questions (e.g., Name, Feed Reason): Enter long and short responses to ensure flexibility.
    • For `integer` (e.g., Years Raising): Enter whole numbers (e.g., 5, 0) and verify no decimals are allowed.
    • For `decimal` (e.g., Feed Cost): Enter numbers with decimals (e.g., 25.50) and confirm they save correctly.
    • For `select_one` (e.g., Gender): Ensure only one option can be selected.
    • For `select_multiple` (e.g., Motivation): Select multiple options and verify all are recorded.
    • For `geopoint` (e.g., Farm Location): Simulate GPS input (if possible) or check that the field prompts for coordinates.
    • For `note` (e.g., Thank You Note): Confirm it displays as text without input fields.
  5. Simulate Full Survey:
    • Complete the survey multiple times with different response patterns (e.g., a farmer currently raising broilers vs. one who stopped).
    • Ensure skip logic paths (e.g., Why Stopped for `stopped_raising` = "Yes") work as expected.
    • Verify the survey ends with the Thank You Note and submits successfully.
  6. Validate Data Output:
    • Submit a few test responses in Preview mode.
    • Go to the project dashboard, click Data, and view the responses.
    • Check that variable names (e.g., `name`, `feed_cost`) match responses and are correctly formatted for export (e.g., CSV).
    • Verify that `select_multiple` responses (e.g., Motivation) are recorded as separate columns or concatenated values, depending on export settings.

Tip: Conduct multiple test runs with varied inputs to catch errors in logic or constraints. Export test data to ensure variable names align with your analysis needs.

6. Deployment and Data Collection

Deploying the Form

Once tested, deploy the form to start collecting data. KoboToolbox supports web and mobile collection, with offline capabilities.

  1. Finalize the Form:
    • In the form builder, ensure all questions, groups, skip logic, and constraints are correct.
    • Save the form and return to the project dashboard.
  2. Deploy the Form:
    • In the project dashboard, locate your form (e.g., "Broiler Farmer Survey").
    • Click the Deploy button (rocket icon).
    • Confirm deployment. The form status will change to "Deployed".
  3. Choose Collection Method:
    • Web Form: Click Open to get a URL for online data collection. Share this link with respondents (e.g., via email or messaging apps).
    • Mobile App (KoboCollect):
      • Download KoboCollect from the Google Play Store or configure it on enumerators’ devices.
      • In the project dashboard, click Settings, then Connect with KoboCollect to get a URL or QR code.
      • Enumerators enter this URL in KoboCollect to download the form for offline use.
  4. Train Enumerators:
    • If using enumerators, train them on the survey structure, question intent, and how to use KoboCollect.
    • Provide guidance on handling skip logic (e.g., skipping Training Details if `training_received` is "No").
    • Test the form on a mobile device to ensure it’s user-friendly in the field.
  5. Monitor Data Collection:
    • Go to the project dashboard and click Data to view incoming responses.
    • Use the Table view to check responses in real-time or download data as CSV for analysis.
    • Monitor for incomplete or erroneous submissions and follow up with enumerators if needed.

Best Practices for Data Collection

  • Pilot Test: Conduct a small-scale pilot with 5–10 respondents to identify issues before full deployment.
  • Offline Considerations: Ensure enumerators sync KoboCollect data when internet is available to avoid data loss.
  • Data Privacy: Inform respondents that their data (e.g., Name, Farm Location) will be handled confidentially. In KoboToolbox Settings, enable encryption if sensitive data is collected.
  • Regular Backups: Download data periodically from the Data tab to prevent loss.

Tip: For mobile data collection in rural areas, ensure enumerators have charged devices and are trained to sync data regularly to avoid delays.

7. Advanced Features

Enhancing the Survey

KoboToolbox offers advanced features to improve survey functionality and data quality. Below are features relevant to the Broiler Farmer Survey.

Adding Calculations

Use `calculate` questions to compute values, such as profit margins based on Feed Cost and Market Price.

  1. Add Calculate Question:
    • In the `market_economics` group, after Market Price, add a question.
    • Type: calculate
    • Label: Profit Margin Calculation
    • Variable Name: profit_margin
    • Calculation: `${market_price} - ${feed_cost} - ${transport_cost}`
  2. Display Result:
    • Add a `note` question after the calculation.
    • Label: Your estimated profit margin per chicken is ${profit_margin} PGK.
    • Variable Name: profit_margin_note
  3. Test Calculation:
    • In Preview, enter sample values (e.g., `market_price` = 25.50, `feed_cost` = 10.00, `transport_cost` = 2.50).
    • Verify the note displays "Your estimated profit margin per chicken is 13.00 PGK."

Multilingual Support

If surveying farmers who speak different languages, add translations.

  1. Enable Multilingual:
    • In the form builder, go to Settings > Form Settings.
    • Add a language (e.g., Tok Pisin).
  2. Add Translations:
    • For each question, click the gear icon and enter translations for Label, Hint, and Choices (e.g., "Name" in Tok Pisin: "Nem").
    • Example for Primary Feed (3.1): "Commercial" → "Komesol", "Home-made" → "Haus-mek", "Mix" → "Miks".
  3. Test Translations:
    • In Preview, switch languages in the dropdown and verify all labels and choices display correctly.

Adding Media

Include images or videos to clarify questions (e.g., pictures of feed types).

  1. Upload Media:
    • In the form builder, go to Settings > Media.
    • Upload an image (e.g., a photo of broiler starter crumbles for question 3.5).
  2. Link to Question:
    • For Starter Used (3.5), click the gear icon, go to Media, and select the uploaded image.
  3. Test Media:
    • In Preview, ensure the image displays with the question on both web and mobile views.

Tip: Use calculations to provide immediate feedback to respondents (e.g., profit margin). For multilingual surveys, work with a native speaker to ensure accurate translations.
