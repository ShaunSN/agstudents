1. Introduction to the Tutorial

What is KoboToolbox?

KoboToolbox is a free, open-source platform designed for data collection in challenging environments, such as agricultural research, humanitarian aid, and public health. It supports digital surveys deployable via mobile devices or web browsers, with offline capabilities and multilingual support. This tutorial uses the web interface, a visual form builder ideal for all skill levels.

2. Setting Up Your Project

Creating a New Project

  1. Log In: Navigate to kobotoolbox.org and sign in.
  2. Start a Project: Click + New Project on the dashboard.
  3. Name It: Enter Broiler Farmer Survey.
  4. Create: Click Create Project.

Opening the Form Builder

  1. Inside the project, select Build from scratch to open the web interface.
  2. Understand the layout:
    • Sidebar (left): Add questions or groups.
    • Main Area: View and edit the form.
    • Settings Panel (right): Configure question properties (e.g., required, skip logic).

Tip: Save your work frequently using the save icon to prevent data loss.

3. Understanding Question Types

Overview

Each question has a type defining its data format and user interaction. Correct selection ensures analyzable data.

  • Text: Open-ended (e.g., Name).
  • Integer: Whole numbers (e.g., Number of broilers).
  • Decimal: Numbers with decimals (e.g., Feed cost in PGK).
  • Select One: Single-choice (e.g., Gender: Male/Female).
  • Select Multiple: Multiple-choice (e.g., Farming motivations).
  • Date: Calendar input (e.g., Chick purchase date).
  • Time: Time input (e.g., Feeding time).
  • Date and Time: Combined input (e.g., Survey timestamp).
  • Note: Non-data text for instructions.
  • Calculate: Computes values (e.g., Profit).
  • Image/Audio/Video: Multimedia (e.g., Farm photo).
  • Geopoint: GPS coordinates (e.g., Farm location).

Choosing the Right Type

  • Use select_one for mutually exclusive options.
  • Use integer/decimal for numerical analysis.
  • Use text for qualitative data.
  • Use geopoint for spatial agricultural data.

Examples:

  • Question: How many broilers are you raising? Type: integer (Reason: Whole number).
  • Question: Average selling price per broiler (PGK)? Type: decimal (Reason: Monetary value, e.g., 5.75 PGK).
  • Question: Diseases affecting your flock? Type: select_multiple, Choices: Newcastle, Coccidiosis, Influenza, Other.

4. Adding Questions to the Survey

Weâ€™ll build key sections from the Broiler Farmer Questionnaire, including Demographics, General Background, Feeding Practices, and Farm Economics.

Demographics Section

  1. Create Group:
    • Click + Add Group.
    • Name: demographics, Label: Demographics.
  2. Add Questions:
    • Name:
      • Type: text
      • Label: What is your name?
      • Variable Name: name
      • Hint: Enter full name of respondent
      • Settings: Check Required.
    • Age:
      • Type: integer
      • Label: How old are you?
      • Variable Name: age
      • Hint: Enter age in years
      • Constraint: ${age} >= 15 and ${age} <= 100
      • Settings: Check Required.
    • Gender:
      • Type: select_one gender
      • Label: What is your gender?
      • Variable Name: gender
      • Choices: male, female
      • Settings: Check Required.
    • Marital Status:
      • Type: select_one marital_status
      • Label: What is your marital status?
      • Variable Name: marital_status
      • Choices: Married, Single, Divorced, Defacto, Rather not say
    • Occupation:
      • Type: select_one occupation
      • Label: What is your main occupation?
      • Variable Name: occupation
      • Choices: Paid Employment, Home Carer, Chicken Farmer, Other

General Background Section

  1. Create Group:
    • Name: general_background, Label: Section 1: General Background.
  2. Add Questions:
    • Farm Location:
      • Type: geopoint
      • Label: Where is your farm located?
      • Variable Name: farm_location
      • Hint: Record GPS coordinates of the farm
    • Years Raising:
      • Type: integer
      • Label: How long have you been raising broiler chickens?
      • Variable Name: years_raising
      • Hint: Enter number of years
      • Constraint: ${years_raising} >= 0
    • Motivation:
      • Type: select_multiple motivation
      • Label: Why did you choose broiler farming?
      • Variable Name: farming_motivation
      • Choices: Cash (Income), For special needs, Low risk, Low capital investment, Quick turnover
    • Training Received:
      • Type: select_one yes_no
      • Label: Have you undergone training before raising chickens?
      • Variable Name: training_received
      • Choices: Yes, No
      • Settings: Check Required.
    • Training Details:
      • Type: text
      • Label: If yes, when and by whom?
      • Variable Name: training_details
      • Skip Logic: selected(${training_received}, 'Yes')

Feeding Practices Section

  1. Create Group:
    • Name: feeding_practices, Label: Section 3: Feeding Practices.
  2. Add Questions:
    • Primary Feed:
      • Type: select_one feed_type
      • Label: What type of feed do you primarily use?
      • Variable Name: primary_feed
      • Choices: Commercial, Homemade, Mix
      • Settings: Check Required.
    • Feed Brand:
      • Type: text
      • Label: What brand of feed(s) do you use?
      • Variable Name: feed_brand
      • Skip Logic: ${primary_feed} = 'Commercial' or ${primary_feed} = 'Mix'
    • Feed Cost:
      • Type: decimal
      • Label: How much does this feed cost per bag (PGK)?
      • Variable Name: feed_cost
      • Constraint: ${feed_cost} > 0
    • Starter Feed:
      • Type: select_one starter_feed_type
      • Label: Do you use broiler starter crumbles?
      • Variable Name: starter_used
      • Choices: Yes, No, Own mix
    • Starter Duration:
      • Type: select_one starter_duration
      • Label: How long do you feed starter crumbles to chicks?
      • Variable Name: starter_duration
      • Choices: 0â€“14 days, 0â€“21 days, 0â€“28 days, 0â€“42+ days
      • Skip Logic: selected(${starter_used}, 'Yes')

Tip: Use the Hint field to provide guidance (e.g., Record GPS coordinates of the farm). Drag questions to reorder for logical flow.

5. Using Advanced Features

Repeat Groups

For repeated data (e.g., details per broiler batch), use repeat groups.

  1. Create a group: batch_details, Label: Broiler Batch Details.
  2. In settings, check Repeat this group.
  3. Add questions:
    • batch_size: Number of chicks in batch? (integer, Constraint: ${batch_size} > 0).
    • start_date: Batch start date? (date).
    • end_date: Batch end date? (date).

Calculations

Use calculate for real-time metrics.

  1. Add a question:
    • Type: calculate
    • Label: Profit calculation
    • Variable Name: profit_calc
    • Expression: ${sell_price} * ${broilers_sold} - ${feed_cost}
  2. Display result:
    • Type: note
    • Label: Your profit is ${profit_calc} PGK.
    • Variable Name: profit_display

Example Calculation: If sell_price = 10 PGK, broilers_sold = 100, and feed_cost = 500 PGK, then profit_calc = 10 * 100 - 500 = 500 PGK.

6. Applying Skip Logic and Constraints

Skip Logic

  1. Example: Broiler Status
    • Question: currently_raising (Are you currently raising broilers?, select_one yes_no, Choices: Yes, No).
    • Question: why_stopped (Why did you stop broiler farming?, select_multiple stop_reason, Choices: No access to feed, High prices of feed, No access to markets, Other).
    • Skip Logic: selected(${currently_raising}, 'No').
  2. In the form builder:
    • Select why_stopped, click the gear icon, go to Skip Logic.
    • Add condition: selected(${currently_raising}, 'No').

Constraints

  • Example: For feed_cost, set ${feed_cost} > 0 to prevent negative values.
  • Apply via the Constraints field in question settings.

Example Skip Logic: If a farmer selects currently_raising = 'No', the why_stopped question appears, allowing them to specify reasons like High prices of feed.

7. Building the Farm Economics Section

  1. Create Group:
    • Name: farm_economics, Label: Section 5: Farm Economics.
  2. Add Questions:
    • income_source: What is your main income source? (select_one income_source, Choices: Broilers, Crops, Other, Required).
    • broiler_income: Annual income from broilers (PGK)? (decimal, Constraint: ${broiler_income} >= 0).
    • feed_cost: Annual feed cost (PGK)? (decimal, Constraint: ${feed_cost} >= 0).
    • profit_calc: Calculate profit (calculate, Expression: ${broiler_income} - ${feed_cost}).
    • profit_display: Your annual profit is ${profit_calc} PGK (note).

Example: If broiler_income = 1000 PGK and feed_cost = 600 PGK, profit_calc displays 400 PGK.

8. Previewing and Testing

  1. Preview Form:
    • Click Preview in the form builder.
    • Test questions, skip logic (e.g., why_stopped appears only if currently_raising = 'No'), and constraints.
  2. Fix Issues:
    • Check for typos, incorrect logic, or missing choices.
    • Reorder questions for logical flow.

Tip: Test with sample responses to ensure calculations (e.g., profit_calc) and skip logic function correctly.

9. Deploying the Survey

  1. Finalize: Click Deploy to activate the form.
  2. Share: Copy the form link for online collection or download for offline use.
  3. Test Deployment: Submit test responses to verify data collection.

Tip: Verify offline functionality if collecting data in areas with limited internet.

10. Best Practices for Survey Design

Design Principles

  • Use simple, direct language (e.g., How many broilers?).
  • Group logically (e.g., demographics before economics).
  • Minimize required fields to maintain respondent goodwill.

Advanced Tips

  • Use calculate for metrics (e.g., cost per broiler: ${feed_cost} / ${broiler_count}).
  • Add note for context (e.g., Next, weâ€™ll ask about feed practices).
  • Test multilingual labels (e.g., English/Tok Pisin for Papua New Guinea).

Pitfalls to Avoid

  • Inconsistent variable names (e.g., feedcost vs. feed_cost).
  • Complex skip logic confusing respondents.
  • Deploying without preview.

Example Pitfall: Using feed_cost in one question and feedcost in a calculation will break the profit_calc expression. Always use consistent variable names.
