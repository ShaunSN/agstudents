Tutorial: Building a Broiler Farmer Survey in KoboToolbox Web Interface

A Comprehensive Step-by-Step Guide with Practical Examples and Advanced Features

Prepared by: Shaun Narimbi

Date: July 9, 2025

1. Introduction to the Tutorial

What is KoboToolbox?

KoboToolbox is a free, open-source platform designed for data collection in challenging environments, such as agricultural research, humanitarian aid, and public health. It supports digital surveys deployable via mobile devices or web browsers, with offline capabilities and multilingual support. This tutorial uses the web interface, a visual form builder ideal for all skill levels.

Purpose of the Tutorial

This tutorial guides you through creating a broiler farmer survey based on the Broiler Farmer Questionnaire. You’ll learn to:

  • Set up a project and form.
  • Add question types (e.g., text, integer, select one/multiple).
  • Organize questions into groups.
  • Apply skip logic, constraints, and calculations.
  • Preview, test, and deploy the survey.

Prerequisites

  • A KoboToolbox account (kobotoolbox.org).
  • Basic familiarity with survey design concepts.

Context

The survey collects data on demographics, general background, feeding practices, economic outcomes, and challenges for broiler farmers.

2. Setting Up Your Project

Creating a New Project

  1. Log In: Navigate to kobotoolbox.org and sign in.
  2. Start a Project: Click + New Project on the dashboard.
  3. Name It: Enter Broiler Farmer Survey.
  4. Create: Click Create Project.

Opening the Form Builder

  1. Inside the project, select Build from scratch to open the web interface.
  2. Understand the layout:
    • Sidebar (left): Add questions or groups.
    • Main Area: View and edit the form.
    • Settings Panel (right): Configure question properties (e.g., required, skip logic).

Tip: Save your work frequently using the save icon to prevent data loss.

3. Understanding Question Types

Overview

Each question has a type defining its data format and user interaction. Correct selection ensures analyzable data:

  • Text: Open-ended (e.g., Name).
  • Integer: Whole numbers (e.g., Age).
  • Decimal: Numbers with decimals (e.g., Feed cost).
  • Select One: Single-choice (e.g., Gender: Male/Female).
  • Select Multiple: Multiple-choice (e.g., Farming motivations).
  • Geopoint: GPS coordinates (e.g., Farm location).
  • Calculate: Computes values (e.g., Profit).
  • Note: Non-data text for instructions.

Choosing the Right Type

  • Use select_one for mutually exclusive options.
  • Use integer/decimal for numerical analysis.
  • Use text for qualitative data.
  • Use geopoint for spatial agricultural data.

Examples

  • Question: How old are you? Type: integer (Reason: Whole number).
  • Question: Normal market price per live chicken? Type: decimal (Reason: Monetary value, e.g., 25.50 PGK).
  • Question: Why did you choose broiler farming? Type: select_multiple, Choices: Cash (Income), Low risk, Quick turnover.

4. Adding Questions to the Survey

We’ll build all sections from the Broiler Farmer Questionnaire, including Demographics, General Background, Chicken Background, Feeding Practices, Nutritional Knowledge, and more.

4.1 Demographics Section

Create Group:

  • Click + Add Group.
  • Name: demographics, Label: Demographics.

Add Questions:

  • Name:
    • Type: text
    • Label: Name
    • Variable Name: name
    • Hint: Enter full name of respondent
  • Age:
    • Type: integer
    • Label: Age
    • Variable Name: age
    • Hint: Enter age in years
  • Place of Origin:
    • Type: text
    • Label: Place of Origin (Home Province)
    • Variable Name: origin
  • Address:
    • Type: text
    • Label: Current address (Home residence/Location)
    • Variable Name: address
  • Gender:
    • Type: select_one gender
    • Label: Gender
    • Variable Name: gender
    • Choices: Male, Female
  • Marital Status:
    • Type: select_one marital_status
    • Label: Marital Status
    • Variable Name: marital_status
    • Choices: Married, Single, Divorced, Defacto, Rather not say
  • Occupation:
    • Type: select_one occupation
    • Label: Main Occupation
    • Variable Name: occupation
    • Choices: Paid Employment, Home Carer, Chicken Farmer, Other
  • Currently Raising:
    • Type: select_one yes_no
    • Label: Are you currently raising broilers?
    • Variable Name: currently_raising
    • Choices: Yes, No
  • Stopped Raising:
    • Type: select_one yes_no
    • Label: Have you raised broilers but stopped?
    • Variable Name: stopped_raising
    • Choices: Yes, No

    • Variable Name: training_received
    • Choices: Yes, No
  • Training Details:
    • Type: text

    • Choices: Friend, Family member/Relative, Neighbour

  • Why Stopped:
    • Type: select_multiple stop_reason
    • Label: 1.6. Why did you stop broiler farming?
    • Variable Name: why_stopped
    • Choices: No access to feed, High prices of feed, No access to day-old chicks, High prices of day-old chicks, No access to markets, Expenses exceeded revenue, Other (explain)

5. Using Advanced Features

Repeat Groups

For repeated data (e.g., details per broiler batch), use repeat groups.

  • Create a group: batch_details, Label: Broiler Batch Details.
  • In settings, check Repeat this group.
  • Add questions:
    • batch_size: Number of chicks in batch? (integer).
    • start_date: Batch start date? (date).

Calculations

Use calculate for real-time metrics.

  • Add a question:
    • Type: calculate
    • Label: Profit calculation
    • Variable Name: profit_calc
    • Expression: ${sell_price} * ${broilers_sold} - ${feed_cost}
  • Display result:
    • Type: note
    • Label: Your profit is ${profit_calc} PGK.
    • Variable Name: profit_display

Example: If sell_price = 10 PGK, broilers_sold = 100, and feed_cost = 500 PGK, then profit_calc = 500 PGK.

6. Applying Skip Logic and Constraints

Skip Logic

Example: Broiler Status

  • Question: stopped_raising (Have you raised broilers but stopped?, select_one yes_no, Choices: Yes, No).
  • Question: why_stopped (Why did you stop broiler farming?, select_multiple stop_reason, Choices: No access to feed, High prices of feed, etc.).
  • Skip Logic: selected(${stopped_raising}, 'yes').
  • In the form builder:
    • Select why_stopped, click the gear icon, go to Skip Logic.
    • Add condition: selected(${stopped_raising}, 'yes').

Constraints

Example: For feed_cost, set ${feed_cost} > 0 to prevent negative values. Apply via the Constraints field.

11. Previewing and Testing

  • Preview Form: Click Preview in the form builder.
  • Test questions, skip logic (e.g., why_stopped appears only if stopped_raising = 'Yes'), and constraints.
  • Fix Issues: Check for typos, incorrect logic (e.g., starter_duration, finisher_start), or missing choices.

Tip: Test with sample responses to ensure skip logic and question flow work as expected.

12. Deploying the Survey

  • Finalize: Click Deploy to activate the form.

  • Test Deployment: Submit test responses to verify data collection.

Tip: Verify offline functionality if collecting data in areas with limited internet.

13. Best Practices for Survey Design

Design Principles

  • Use simple, direct language (e.g., "How much does this feed cost?").
  • Group logically (e.g., demographics before feeding practices).
  • Minimize required fields to maintain respondent goodwill.

Advanced Tips

  • Use select_multiple for questions with multiple answers (e.g., "What type of other feedstuff do you use?"); data is stored as space-separated values (e.g., purchased kitchen_waste).
  • Add note for context (e.g., "Next, we’ll ask about nutritional knowledge").
  • Test multilingual labels if applicable.

Pitfalls to Avoid

  • Inconsistent variable names (e.g., feedcost vs. feed_cost).
  • Complex skip logic confusing respondents.
  • Deploying without preview.
