1. Introduction to the Tutorial

Understanding KoboToolbox and its capabilities for agricultural research

What is KoboToolbox?

KoboToolbox is a free, open-source platform designed for data collection in challenging environments, such as agricultural research, humanitarian aid, and public health. It supports digital surveys deployable via mobile devices or web browsers, with offline capabilities and multilingual support. This tutorial focuses on the web interface, a user-friendly form builder suitable for beginners and advanced users alike.

Purpose of the Tutorial

This tutorial guides you through creating a comprehensive broiler farmer survey based on the Broiler Farmer Questionnaire. By the end, you'll be able to:

  • Set up a project and form in KoboToolbox
  • Add various question types (text, integer, select_one, select_multiple)
  • Organize questions into logical groups for better survey flow
  • Implement skip logic, constraints, and calculations for dynamic surveys
  • Preview, test, and deploy the survey for data collection

Prerequisites

To follow this tutorial, you need:

  • A KoboToolbox account (sign up at kobotoolbox.org)
  • Basic understanding of survey design concepts
  • Access to a web browser for the KoboToolbox web interface

Context

The survey collects data on broiler farmers' demographics, general background, chicken rearing practices, feeding strategies, nutritional knowledge, economic considerations, challenges, performance outcomes, and future improvement suggestions. This data is critical for agricultural research and support programs.

3. Understanding Question Types

Choosing the right question type for optimal data collection

Overview

KoboToolbox offers various question types to capture different data formats. Choosing the correct type ensures data is collected in a format suitable for analysis.

Text

For open-ended responses (e.g., respondent's name)

Integer

For whole numbers (e.g., age in years)

Decimal

For numbers with decimals (e.g., feed cost in PGK)

Select One

For single-choice questions (e.g., Gender: Male or Female)

Select Multiple

For multiple-choice questions (e.g., reasons for choosing farming)

Geopoint

For capturing GPS coordinates (e.g., farm location)

Calculate

For computing values based on other inputs (e.g., profit)

Note

For non-data text, such as instructions or context

Choosing the Right Type

Use select_one for mutually exclusive options to ensure single responses

Use integer/decimal for numerical data suitable for calculations

Use text for qualitative, descriptive responses

Use geopoint for spatial data, critical for agricultural surveys

Examples

Question: How old are you? Type: integer (Age is a whole number, e.g., 45)

Question: Normal market price per live chicken? Type: decimal (Monetary values may include decimals, e.g., 25.50 PGK)

Question: Why did you choose broiler farming? Type: select_multiple (Allows multiple reasons like Cash, Low risk, Quick turnover)

Group Label: Demographics

Hint: Enter age in years

Type: text

Variable: address

Label: Current address (Home residence/Location)

Gender

Type: select_one

Variable: gender

Choices: Male, Female

Marital Status

Type: select_one

Variable: marital_status

Choices: Married, Single, Divorced, Defacto, Rather not say

Main Occupation

Type: select_one

Variable: occupation

Choices: Paid Employment, Home Carer, Chicken Farmer, Other

Currently Raising

Type: select_one

Variable: currently_raising

Choices: Yes, No

Implementation Instructions

  1. Create Group: Click + Add Group, set Name to demographics and Label to Demographics
  2. Add Questions: For each question, select appropriate type, enter label, set variable name, add choices for select_one questions
  3. Add Constraints: Optional constraint for Age: ${age} >= 18
  4. Test: Use Preview mode to test question flow and validate inputs

4.2 General Background Section

Group Name: general_background

Group Label: Section 1: General Background

1.1 Farm Location

Type: geopoint

Variable: farm_location

Label: Where are you located (location of farm)?

Hint: Record GPS coordinates of the farm

1.2 Years Raising

Type: integer

Variable: years_raising

Label: How long have you been raising broiler chickens?

Constraint: ${years_raising} >= 0

1.3 Farming Motivation

Type: select_multiple

Variable: farming_motivation

Label: Why did you choose broiler farming?

Choices: Cash (Income), For special needs, Low risk, Low capital investment, Quick turnover

1.4 Training Received

Type: select_one

Variable: training_received

Label: Have you undergone training before raising chickens?

Choices: Yes, No

1.5 Training Details

Type: text

Variable: training_details

Label: If yes, when and by who?

Skip Logic: selected(${training_received}, 'yes')

1.6 Introduction Source

Type: select_one

Variable: introduction_source

Label: Who introduced broiler chicken farming to you?

Choices: Friend, Family member/Relative, Neighbour

Skip Logic: selected(${training_received}, 'no')

1.7 Why Stopped

Type: select_multiple

Variable: why_stopped

Label: Why did you stop broiler farming?

Choices: No access to feed, High prices of feed, No access to day-old chicks, High prices of day-old chicks, No access to markets, Expenses exceeded revenue, Other (explain)

Skip Logic: selected(${stopped_raising}, 'yes')

Skip Logic Implementation

• Training Details: Shows only if training was received

• Introduction Source: Shows only if no formal training

• Why Stopped: Shows only if respondent stopped raising chickens

• Test all conditions thoroughly in Preview mode

5. Testing and Validation

Ensuring your survey functions correctly before deployment

Why Test?

Testing ensures the survey functions as intended, skip logic works correctly, constraints prevent invalid data, and the overall user experience is smooth. Validation confirms data is collected accurately and in the expected format.

Preview the Form

  1. 1 Click the Preview button (eye icon) in the form builder
  2. 2 Navigate through all sections to check layout and flow
  3. 3 Verify question labels, hints, and choice options display correctly
  4. 4 Check that group labels organize questions logically

Test Skip Logic

  1. 1 Test Training Details: Set training_received to "Yes" (should show), then "No" (should hide)
  2. 2 Test Introduction Source: Set training_received to "No" (should show), then "Yes" (should hide)
  3. 3 Test all conditional questions in feeding practices section
  4. 4 Verify questions appear/disappear based on conditions

Test Constraints

  1. 1 Try entering negative values for Age, Years Raising, costs
  2. 2 Verify error messages appear for invalid inputs
  3. 3 Test decimal inputs accept numbers with decimal points
  4. 4 Confirm survey cannot be submitted with invalid data

Test Question Types

  1. 1 Test text fields accept various characters and lengths
  2. 2 Test integer fields only accept whole numbers
  3. 3 Test select_one allows single selection only
  4. 4 Test select_multiple allows multiple selections
  5. 5 Test geopoint captures coordinates (if on mobile device)

Simulate Full Survey

Create Different Respondent Profiles:

  • • New farmer (0 years experience)
  • • Experienced farmer (5+ years)
  • • Farmer who stopped raising chickens
  • • Farmer with formal training
  • • Farmer without formal training

Complete entire survey for each profile to ensure all paths work correctly.

Validate Data Output

  1. 1. Submit test responses
  2. 2. Export data to Excel/CSV
  3. 3. Check column headers match variable names
  4. 4. Verify data types are correct
  5. 5. Confirm skip logic results in blank cells where appropriate
  6. 6. Test multiple choice questions show selected options

Testing Tips

• Test on Different Devices: Preview on mobile, tablet, and desktop to ensure responsive design

• Test Internet Connectivity: If using mobile data collection, test offline functionality

• Involve Others: Have colleagues test the survey to identify issues you might miss

• Time the Survey: Complete the survey to estimate how long it takes respondents

• Document Issues: Keep a list of problems found during testing for systematic fixing

Enhancing the Survey

Beyond basic questions, KoboToolbox offers advanced features to make your survey more dynamic and informative. These features include automatic calculations, multilingual support, and media integration.

7.1 Adding Calculations

Automatically compute values based on user inputs to provide immediate feedback and insights.

Example: Profit Margin Calculation

Step 1: Add Calculate Question

Type: calculate

Variable: profit_margin

Calculation: ${sell_price} - ${feed_cost} - ${transport_cost} - ${price_chicks}

Step 2: Display Result

Type: note

Label: Your estimated profit margin per chicken is: ${profit_margin} PGK

Implementation Steps

  1. 1
    Add Calculate Question:

    Click + Add Question, select "calculate" type

  2. 2
    Enter Formula:

    Use ${variable_name} format to reference other questions

  3. 3
    Add Display Note:

    Create a note question to show the calculated result

  4. 4
    Test Calculation:

    Preview the form and enter test values to verify calculation
